Projects

Here are my currently active projects, in no particular order. When no collaborators are listed, it can be assumed that Stephanie has her hand in the project.

  • Make Haskell dependently typed. We propose an update to FC, the internal language of GHC, that brings kind equalities to the language. The new System FC merges types and kinds and has the (*:*) axiom. The paper (and extended version), was published at ICFP 2013. I have implemented these ideas into GHC. Ongoing work includes working out a type inference algorithm in order to carry our changes from the internal language into Haskell. With Stephanie Weirich.
  • Roles in GHC. I have implemented roles into GHC, as released as part of GHC 7.8. Here is a draft paper, accepted to ICFP 2014, on the subject. (extended version) There continues to be ongoing debate about roles and the best way forward with them, and I expect to implement whatever new features are necessary.
  • singletons. This project supports the singletons package, allowing a limited form of dependently typed programming in current versions of Haskell. The newest version of singletons supports the promotion of a wide array of term-level functions to type families. See the draft paper. The original paper resulting from this work was published at the Haskell Symposium, 2012. With Jan Stolarek.

Below are my less active projects. Most weeks do not involve any work on these, but issues crop up now and again.

  • Roles. I implemented a simplified version of the ideas in Generative type abstraction and type-level computation into GHC. This prevents the 
    GeneralizedNewtypeDeriving
    bug. See also the wiki page on the subject. This work was completed while I was preforming an internship at Microsoft Research Cambridge, under the supervision of Simon Peyton Jones.
  • units. This is a library, written using closed type families, that allows for type-level dimensional analysis, to aid programs written using units-of-measure. The project page is here.
  • Closed type families in Haskell. What changes are necessary to the surface language and in GHC to allow closed type families with ordered, overlapping equations? See the GHC wiki page for more info. This feature is now part of GHC and will be included in the next major release. We have also submitted a paper (extended version, with proofs and further examples) to POPL 2014. With Dimitrios Vytiniotis, Simon Peyton Jones, and Stephanie Weirich.
  • System FC in GHC. I maintain a formal specification of the internal language within GHC here.
  • Ironclad C++. We define a subset of C++ that can be proved not to allow access to invalid memory locations. Our paper will be presented at OOPSLA 2013. Here is the accompanying technical report, with the proof of the critical pointer-lifetime invariant. With Christian DeLozier, Peter-Michael Osera, Santosh Nagarakatte, Milo M. K. Martin, and Steve Zdancewic.
